Ya expliqué en posts anteriores cómo iniciar un proyecto de Godot con Github usando un template que hice, y cómo probar el juego de Godot exportado a HTML5 de manera local, ahora viene cómo publicarlo y jugarlo desde Github Pages.

Github Pages

Github permite usar un repositorio como un limitado hosting para una página web totalmente gratuito, incluso, podemos crear una página web estática desde cero y usar un dominio propio con Github Pages, pero aquí no vamos a hacer eso, vamos a usar el repositorio de nuestro proyecto en github para poder visualizar desde la web el juego.

#1 Crea una carpeta "docs"

Luego de crear la carpeta docs debemos exportar el juego, seleccionamos HTML5 en Godot y lo nombramos index.html y lo guardamos en dicha carpeta.

#2 Configuramos el repositorio

Luego de actualizar el repositorio (git add ., git commit, etc.) vamos a la web de Github y ubicamos el repositorio, nos dirigimos a Settings y bajamos hasta encontrar la opción Source, ahí debemos cambiar la opción a master branch /docs folder, como aparece en la imagen.


Una vez seleccionamos, podremos usar el link que nos indica para poder ver el juego online.

¡Listo! Ya puedes compartir tu enlace para que otras personas puedan jugar a tu juego.

Incrustar el juego en una página web

Adicionalmente puedes usar Github Pages para incrustar el juego a otros sitios usando la etiqueta <iframe>. El formato a seguir sería el siguiente:

<iframe src="link_github.html" width=1024 height=600></iframe>

Como ejemplo te dejo incrustado un juego mío:

Código:
